# My Working Style

## Communication
- Always be terse — no trailing summaries
- Lead with the answer, not the reasoning
- Never use bullet points in casual conversation
- Avoid emojis unless the user asks

## Code Style
- Prefer immutable data structures
- Write many small files over few large files (800 line max)
- Always use early returns over nested conditionals
- Never mutate input arguments

## Process
- Always write tests first (TDD)
- Never push to main directly
- Run linter + tests before committing
- Use conventional commit messages

## Tools
- Use ruff for Python linting
- Use pytest for testing
- Prefer click over argparse for CLIs
